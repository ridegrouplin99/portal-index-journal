# portal-index-journal

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面集合的存储与索引。

## 项目简介

本仓库旨在提供一种轻量级的方式，将多个独立编写的 HTML 页面集中存放，并通过 GitHub Pages 等静态托管服务进行发布。每个页面均为自包含的 HTML 文件，彼此独立，互不依赖。

适合场景包括但不限于：
- 个人实验性页面存档
- 小型活动或项目的信息页集合
- 临时展示页面的集中管理

## 目录结构

```
portal-index-journal/
├── index.html          # 主索引页面（可选）
├── pages/              # 存放各个独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（样式、图片、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立 HTML 页面存放于此目录，每个文件对应一个页面。
- `assets/`：存放页面间共享的静态资源，便于统一管理样式、脚本和图片。
- `index.html`：可选的根目录索引文件，用于提供页面导航或列表。

## 页面归档说明

- 每个 HTML 页面应为完整、自包含的文档（包含 `<html>`、`<head>`、`<body>` 等基本结构）。
- 页面之间无硬编码链接依赖，除非通过索引页进行导航。
- 资源引用建议使用相对路径，以确保在本地和部署后均可正常访问。
- 建议在页面内部使用 `<meta>` 标签注明标题、描述等信息，便于检索。

## 维护说明

- 新增页面：直接在 `pages/` 目录下创建新的 `.html` 文件。
- 更新页面：修改对应文件后提交即可，无需额外配置。
- 删除页面：从 `pages/` 目录移除文件，并检查索引页（如有）是否需要更新。
- 资源更新：如需修改公共样式或脚本，请更新 `assets/` 下对应文件，注意兼容性。

## 部署

本仓库默认启用 GitHub Pages 功能，发布源设置为 `main` 分支根目录。部署后，可通过仓库对应的 Pages 链接访问 `index.html` 或 `pages/` 下的各个页面。

如需自定义域名或部署方式，请参考 GitHub Pages 官方文档进行配置。

## 注意事项

- 本仓库不追踪、不索引外部网站内容，仅作为独立页面的容器。
- 请勿在页面中嵌入任何违反平台政策或法律法规的内容。
- 页面内容由贡献者自行负责，仓库维护者不承担内容审核义务。
